# Specifications

Feature specifications generated by `/speckit:specify` live here, one directory per feature.

## Layout

Each feature has a folder like `001-feature-name/` containing:

| File | Produced by | Contains |
|---|---|---|
| `spec.md` | `/speckit:specify` | What the feature should do (no how) |
| `plan.md` | `/speckit:plan` | How to build it; references our architecture |
| `tasks.md` | `/speckit:tasks` | Ordered, atomic tasks — each independently executable |
| `research.md` | `/speckit:plan` | Open questions + research findings (optional) |
| `data-model.md` | `/speckit:plan` | Domain types/state shapes for the feature (optional) |
| `contracts/*.md` | `/speckit:plan` | Port interfaces the feature declares (optional) |
| `checklists/*.md` | `/speckit:checklist` | Review checklists applied during implementation |

## Workflow

```
 /speckit:specify  →  spec.md
        │
        ▼
 /speckit:clarify  →  resolves [NEEDS CLARIFICATION] markers
        │
        ▼
 /speckit:plan     →  plan.md (+ research.md, data-model.md, contracts/)
        │
        ▼
 /speckit:tasks    →  tasks.md
        │
        ▼
 /speckit:analyze  →  cross-check spec ⇄ plan ⇄ tasks for gaps
        │
        ▼
 /speckit:implement → walk through tasks, one at a time
```

Every step references `.specify/memory/constitution.md` and must align with the
rules in `docs/ai/architecture.md` and `docs/ai/contribution-contract.md`.

## The constitution

`.specify/memory/constitution.md` is the authoritative rule set referenced by
every spec-kit command. It points to our existing architecture + contribution
docs so spec-kit and the rest of the AI-first scaffolding agree.
