# Learn Game Dev — for Programmers Who Never Had the Time

You can already write production code. You've built APIs, services, websites. You've wanted to make a game for years but kept putting it off — the art, the audio, the "I don't know where to start" feeling. This folder exists to close that gap in the fewest pages possible, so this template can be the difference between "one day" and "next weekend."

**Who this is for:**
- Tech leads, backend engineers, web devs, systems engineers
- People who can ship software but have never shipped a game
- Anyone who wants to become a **one-person game studio** with AI leverage

**Who this is _not_ for** (you already know this stuff):
- Professional game developers
- Trained illustrators / animators / musicians
- Anyone who's already shipped a commercial game

## The one-line version of each doc

If you only have five minutes, here's the condensed version:

1. A game is an infinite loop that reads input, updates state, and renders — once per frame. Your domain logic stays pure; Phaser is the I/O shell.
2. Sprites are images, tilemaps are grids of images, collisions are box overlaps, animation is a sequence of images shown over time. That's 80% of what you need.
3. You don't need to draw. Midjourney, Scenario, Retro Diffusion, Suno, ElevenLabs cover art, animation, music, and sound effects respectively. Spend 30 minutes learning each and you have a full pipeline.
4. A good game is one mechanic that's fun to do for 90 seconds, repeated with slowly-increasing challenge. Everything else is decoration.
5. Prototype in a weekend. If it isn't fun at 10 minutes old, no amount of polish will fix it. Ship small, ship often, ship on itch.io.
6. The best tutorials are GameMaker's Toolkit (YouTube), the Phaser Examples site, and the book *The Art of Game Design*.

Go build something.
